* Indicators
  We describe here how the indicators are computed and what are the AC
  codes involved in their computation.  For each indicator, we give a
  table with four columns. The first column has the ISCED level. The
  second one has the alphanumeric code used for the indicator. The
  third column has the formula used to compute this indicator. The
  fourth column contains yes or no depending on whether the AC code
  has a highest and lowest indicator among all the regions.

  The superscript L, means that the value from the previous year is
  being taken. For example, if the emco year is 2015 T.1^{L} means
  that the 2014 value of T.1.

  There are some alphanumeric codes that do not appear in the file
  =Indic_IDs.xlsx= in Dropbox. For these ones we created new AC
  codes. The super script n will be used in the table when this is the
  case.

  There are some AC codes in the file =Indic_IDs.xlsx= that we believe
  are wrong. The most common case is for the AC code to have the .Pu
  dimension to denote a total instead of only the private
  schools. When this is the case the superscript * is used in the
  table. In this cases there is an explanation under the table
  explaining the reason for the modification.

  Some indicators correspond to the highest and lowest value among all
  the regions of some indicator. No AC codes appear in
  =Indic_IDs.xlsx= for these indicators. Thus, we have convened that
  if A is some alphanumeric code. H.A and L.A correspond to the
  highest and lowest value, respectively, among all the regions for
  the AC code A. The fourth column of each indicators says if there is
  also a highest and lowest indicator .

** Mean age 
   + Python method :: mean_age_level(level in ['T.1', 'T.2.GPV', 'T.3.GPV', 'T.23.GPV'])
   + AC_CODES information :: 
        |          | AC_CODE       | FORMULA                                                           | H & L |
        |----------+---------------+-------------------------------------------------------------------+-------|
        | ISCED 1  | MAge.T.1      | $\sum_{\text{ y in cat}} T.1.y \times MidDur(y) / Pop.Ag0t99$     | No    |
        | ISCED 2  | MAge.T.2.GPV  | $\sum_{\text{ y in cat}} T.2.GPV.y\times MidDur(y) / Pop.Ag0t99$  | No    |
        | ISCED 3  | MAge.T.3.GPV  | $\sum_{\text{ y in cat}} T.3.GPV.y\times MidDur(y) / Pop.Ag0t99$  | No    |
        | ISCED 23 | MAge.T.23.GPV | $\sum_{\text{ y in cat}} T.23.GPV.y\times MidDur(y) / Pop.Ag0t99$ | No    |
        |----------+---------------+-------------------------------------------------------------------+-------|
        where ~Cat~ is ~Ag20m~, ~Ag20t29~, ~Ag30t39~, ~Ag40t49~, ~Ag50t59~ and ~Ag60p~. Midpoints are based on the following:
        | Age in years | <20 | 20-29 | 30-39 | 40-49 | 50-59 | 60+ |
        |--------------+-----+-------+-------+-------+-------+-----|
        | Midpoint     |  20 |  24.5 |  34.5 |  44.5 |  54.5 |  60 |
        |--------------+-----+-------+-------+-------+-------+-----|
        Note all indicators are calculated for Total, Public and Private levels.
          
** Mean years of experience 
   + Python method :: mean_exp_level(level in ['T.1', 'T.2.GPV', 'T.3.GPV', 'T.23.GPV'])
   + AC_CODES information :: 
        |          | AC_CODE       | FORMULA                                                           | H & L |
        |----------+---------------+-------------------------------------------------------------------+-------|
        | ISCED 1  | MExp.T.1      | $\sum_{\text{ y in cat}} T.1.y \times MidDur(y) / Pop.Ag0t99$     | No    |
        | ISCED 2  | MExp.T.2.GPV  | $\sum_{\text{ y in cat}} T.2.GPV.y\times MidDur(y) / Pop.Ag0t99$  | No    |
        | ISCED 3  | MExp.T.3.GPV  | $\sum_{\text{ y in cat}} T.3.GPV.y\times MidDur(y) / Pop.Ag0t99$  | No    |
        | ISCED 23 | MExp.T.23.GPV | $\sum_{\text{ y in cat}} T.23.GPV.y\times MidDur(y) / Pop.Ag0t99$ | No    |
        |----------+---------------+-------------------------------------------------------------------+-------|
        where ~cat~ is  ~Exp1t2~,  ~Exp3t5~, ~Exp6t10~,  ~Exp11t15~ and  ~Exp15p~. 
       ~MidDur~ is ~[Exp1t2 = 1.5, Exp3t5 = 4, Exp6t10 = 8, Exp11t15 = 13, Exp15p = 15]~. Mid durations are based on the following:  
        | experience in years | New | 1-2 | 3-5 | 6-10 | 11-15 | 15+ |
        |---------------------+-----+-----+-----+------+-------+-----|
        | Mid duration        | 0.5 | 1.5 |   4 |    8 |    13 |  17 |
        |---------------------+-----+-----+-----+------+-------+-----|     
        Note all indicators are calculated for Total, Public and Private levels.
